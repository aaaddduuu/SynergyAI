### 3. 优化 Agent 提示词 - 强调增量工作原则 🎯

**问题**: Agent 倾向于一次性完成所有工作，导致质量下降

**解决方案**:
- ✅ 添加 `INCREMENTAL_WORK_PRINCIPLES` 常量（基于 Anthropic 文章）
- ✅ 为所有 Agent 角色添加增量工作指导
- ✅ 强调"一次只处理一个任务"
- ✅ 强调"确保真正完成"（测试驱动）

**修改文件**:
- `core/agents.py:54-94` - 添加增量工作原则
- `core/agents.py:97-260` - 更新所有 Agent 系统提示

**增量工作原则核心**:
1. **一次只处理一个任务** - 避免上下文混乱
2. **确保真正完成** - 已实现、已测试、可用
3. **留下干净状态** - 其他人可以立即继续
4. **查看进度摘要** - 快速恢复上下文
5. **测试驱动验证** - 不只看代码，要真正测试

**收益**:
- ✅ Agent 工作更有条理
- ✅ 减少半成品和遗漏功能
- ✅ 提高任务完成质量
- ✅ 更好的协作效率

**新增文件**:
- `core/testing.py` (420 行) - 测试验证框架

**核心功能**:
```python
# 快速验证示例
from core.testing import quick_validate

# 验证非空
is_valid, message = quick_validate(value, "not_empty", field_name="任务标题")

# 验证长度
is_valid, message = quick_validate(text, "length", min_len=10, max_len=200)

# 验证包含关键词
is_valid, message = quick_validate(content, "contains", keyword="登录")
```

**QA Agent 测试流程**:
```
1. 理解任务需求和验收标准
2. 列出测试要点（至少3个）
3. 对每个要点进行验证：
   - 测试项：xxx
   - 测试方法：xxx
   - 测试结果：✅通过 / ❌失败
   - 问题说明：（如果有）
4. 给出最终结论：通过 / 不通过
```

**收益**:
- ✅ QA Agent 有明确的测试方法
- ✅ 可以验证任务完成情况
- ✅ 减少虚假完成任务
- ✅ 提高交付质量
