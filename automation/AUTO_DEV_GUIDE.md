# 🤖 SynergyAI 自动化开发循环使用指南

## 📋 概述

`run_dev_loop.sh` 是一个自动化脚本，用于循环调用 Claude Code 执行完整的开发流程。每次循环都会：
1. 启动/检查开发服务器
2. 调用 Claude Code 执行开发任务
3. 自动提交代码变更
4. 记录详细日志

---

## 🚀 快速开始

### Linux/macOS

```bash
# 1. 给脚本添加可执行权限
chmod +x run_dev_loop.sh

# 2. 运行脚本（执行 5 次开发循环）
./run_dev_loop.sh 5
```

### Windows

```cmd
REM 运行批处理脚本（执行 5 次开发循环）
run_dev_loop.bat 5
```

## ⚙️ 配置说明

### 脚本参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `循环次数` | 必需，指定执行多少次开发循环 | `./run_dev_loop.sh 10` |

### 环境变量（可选）

可以在脚本开头修改以下配置：

```bash
# 开发服务器端口
SERVER_PORT=8000

# Claude Code CLI 命令
CLAUDE_CMD="claude"

# 初始 Prompt（每次循环都会发送给 Claude）
INITIAL_PROMPT="请继续 SynergyAI 项目的开发工作..."
```

### Claude Code 权限参数

脚本使用以下参数确保自动化运行：

| 参数 | 说明 |
|------|------|
| `--yes` | 自动确认所有提示 |
| `--no-interactive` | 非交互模式 |
| `--allow-permissions` | 允许所有权限（文件读写、执行命令等） |

## 📊 日志系统

### 日志文件位置

所有日志保存在 `logs/` 目录：

```
logs/
├── dev_loop_20250321_143022.log       # 主日志文件
├── server_20250321_143025.log          # 服务器日志
├── claude_1_20250321_143105.log        # 第1轮 Claude 日志
├── claude_2_20250321_143520.log        # 第2轮 Claude 日志
└── ...
```

### 日志级别

- `[INFO]` - 一般信息
- `[SUCCESS]` - 成功操作
- `[WARNING]` - 警告信息
- `[ERROR]` - 错误信息
- `[STEP n/m]` - 进度步骤

## 🔄 工作流程

### 单次循环流程

```
┌─────────────────────────────────────┐
│  1. 检查服务器状态                  │
├─────────────────────────────────────┤
│  2. 调用 Claude Code                │
│     - 发送初始 Prompt               │
│     - 执行开发任务                  │
│     - 运行测试                      │
├─────────────────────────────────────┤
│  3. 检查代码变更                   │
├─────────────────────────────────────┤
│  4. 提交变更 (git commit)           │
├─────────────────────────────────────┤
│  5. 推送到远程 (git push)           │
├─────────────────────────────────────┤
│  6. 等待 5 秒，进入下一轮          │
└─────────────────────────────────────┘
```

## 🛠️ 故障排除

### 问题 1: 服务器启动失败

**症状**: `服务器启动超时`

**解决方案**:
```bash
# 1. 检查端口占用
lsof -i :8000

# 2. 终止占用端口的进程
kill -9 <PID>

# 3. 手动启动服务器测试
python main.py
```

### 问题 2: Claude Code 命令不存在

**症状**: `claude: command not found`

**解决方案**:
```bash
# 1. 检查 Claude Code CLI 是否安装
which claude

# 2. 如果未安装，请安装 Claude Code
# 参考: https://claude.ai/code

# 3. 或者修改脚本中的 CLAUDE_CMD 变量指向完整路径
```

### 问题 3: Permission 提示

**症状**: Claude Code 仍然请求权限确认

**解决方案**:
```bash
# 检查脚本中的 Claude 参数
# 确保 --allow-permissions 参数存在
# 或者使用 Claude Code 的配置文件设置默认权限
```

### 问题 4: Git 推送失败

**症状**: `代码已推送到远程仓库` 后面显示 `推送失败`

**解决方案**:
```bash
# 1. 检查远程仓库地址
git remote -v

# 2. 检查认证配置
git config --global credential.helper

# 3. 手动推送测试
git push origin main
```

## ⚠️ 注意事项

1. **确保 Claude Code CLI 已安装**并配置好 API 密钥
2. **确保 Python 环境正常**，依赖已安装
3. **确保 Git 配置正确**，可以正常推送
4. **建议在测试环境先运行**，确认流程正常
5. **定期检查日志**，确保开发质量
6. **注意 API 配额**，避免超出限制
